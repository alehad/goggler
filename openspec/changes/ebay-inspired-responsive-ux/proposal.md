# Change: eBay-inspired responsive UX

## Why

goggler should feel familiar to users who already use eBay on iPhone, iPad, Mac, or web. The app should borrow broad marketplace conventions such as a prominent search surface, activity-oriented navigation, watchlist/purchase-history language, compact listing rows, filters, and account/preferences organization.

This is also the right time to shape the web app so it can later become an iPhone app without rethinking the information architecture.

## What Changes

- Reframe the main navigation around marketplace activity: Home, Watching, Purchases, and My goggler.
- Use a persistent bottom navigation dock so the web shell maps closely to a future iPhone app.
- Make item/history layouts denser and closer to marketplace listing rows, with thumbnails, title, price context, seller, condition, time left, and quick actions.
- Make the Account area feel like a preferences/account hub rather than a generic settings panel.
- Keep goggler visually distinct from eBay branding while remaining familiar in interaction patterns.

## Out Of Scope

- Copying eBay branding, exact screens, icons, or proprietary visual design.
- Native iOS implementation.
- Real imported eBay item data.
- Buying-history import behavior.
- Push notifications or native device integration.

## Success Criteria

- The web shell uses a navigation model that maps naturally to iPhone bottom tabs.
- The dashboard uses compact, scannable listing rows for likely relistings.
- Watching and Purchases language aligns with eBay buyer mental models.
- The Account tab groups local sign-in, eBay connection, and preferences in a familiar account/settings layout.
- The UI remains responsive and usable on mobile and desktop.
